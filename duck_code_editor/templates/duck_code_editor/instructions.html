Instructions

  1. Please read and follow the instructions before each task! The results of our experiment hinge on it.

  2. Each task is a Python function that compiles correctly but produces incorrect or unexpected output. Hence, your goal is to debug these pieces of code. All bugs are either

    • short lines of code that are missing to make the algorithm work, or

    • errors in code logic (e.g., incorrect manipulation of variables, wrong indexing, incomplete conditions, ...)

    You will NOT have to add or remove any control flow structures (for-loops, if statements, ...) in ANY of the tasks. You MIGHT, however, have to edit their conditions.

  3. Each task will be introduced by a description of the code, some example input/output. Your output after debugging should exactly match that of the examples.

  4. Do NOT use ANY external tools (ChatGPT, web browser, …) during debugging. You may only rely on your own knowledge. All bugs can be solved with basic Python statements.

  5. Do NOT import new libraries. Any needed libraries will already be included.

  6. Do NOT define new functions. Only edit any existing ones.

  7. Do NOT refresh the page or use the back button in your browser.

  8. When you think your code is corrected, or when you decide you cannot fix it, please press next. You will not be able to return after proceeding, so make sure you are certain that you are done before moving on.

  9. There is a 10-minute time limit on each task.
